# Meeting - 20260723

## Completed Work

- Fixed an 8-byte alignment bug in BPFtime software perf records that caused reader spinning and silent event loss.

- Corrected the cost attribution: the full probe cost must include both the nginx worker and the `sslsniff` reader. The remaining Jetson overhead is mainly on the nginx side.

## Key Results

### Impact of the Alignment fix

| Metric                    |   Before |   After |      Change |
| ------------------------- | -------: | ------: | ----------: |
| Reader instructions       | 22.032 B | 0.866 B | **-96.07%** |
| Mean bpftime benchmark CV |   14.21% |   2.57% | **-81.94%** |

### Nginx-Side Tracing Cost

| Platform | Payload and metric                        |    Kernel |   bpftime | bpftime / Kernel |
| -------- | ----------------------------------------- | --------: | --------: | ---------------: |
| Jetson   | 16 B, instructions/request                |    25.1 k |    46.0 k |        **1.83x** |
| x64      | 16 B, CPU-time/request, global attach     | 26.891 us | 15.221 us |       **0.566x** |


Jetson and x64 use different performance counters, so only the within-platform ratios are compared.

## Current Conclusion

- CPU affinity save, pin, and restore account for about **70%** of each <font color="#ff0000">bpftime event-output</font> helper call.

- Event-output calls increase from **1.98 to 17.92 per request** between 16 B and 256 KB, larger payloads repeatedly pay this fixed per-event cost, offsetting the savings from avoiding the kernel tracing path.

- The result is platform-dependent: bpftime remains faster than kernel eBPF in the x64 benchmark.(about 20~30%)
